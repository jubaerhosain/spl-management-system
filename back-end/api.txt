GET: query param or param (never use body)
POST, PUT: always use body 
DELETE: try to use body but some cases you can use param to locate resource

1. Authentication 
    post: auth/login
    get: auth/ {authenticated user}
    delete: auth/logout

    put: auth/change-password
    put: auth/generate-otp
    put: auth/verify-otp
    put: auth/reset-password

2. Student
    post: student/
    put: student/ {by himself or admin will be business logic}
    get: student/{studentId}
    get: student/ {query params: optional}
    delete: student/{studentId} {by admin}

2. Teacher
    post: teacher/
    put: teacher/ {by himself or admin will be business logic}
    get: teacher/{teacherId}
    get: teacher/ {query params: optional}
    delete: teacher/{teacherId} {by admin}

3. SPL 
    post: spl/
    put: spl/
    get: spl/{splId|splName} 
    get: spl/ {query params: optional}
    delete: spl/{splId|splName}

    post: spl/student-allocation/ body{splId|splName} // create mark table also for each student
    delete: spl/student-allocation/  body{splId|splName, studentId} // delete mark table also for that student

    put: spl/member/ body{splId|splName}  
    delete: spl/member/ body{splId|splName, memberId}

4. Team
    post: team/
    put: team/
    get: team/{teamId}
    get: team/{params: optional, splName, splId}
    delete: team/{teamId}

    put: team/member
    delete: team/member/{memberId}

5. Request
    put: request/team/ body{teamId, teacherId}
    delete: request/team/ body{teamId, teacherId}

    put: request/student/ {studentId: from logged in user}
    delete: request/student/ body{teacherId} {studentId: from logged in user}

4. Supervisor
    put: supervisor/ body{randomization}  body{teacherId, studentId, splId/splName}
    delete: supervisor/ body{studentId, splId/splName}

5. Mark 
    put: mark/{markId} update atomic marks 
    get: mark/{markId} 
    get: mark/ {query params: splId|splName} mark of all student for a specific spl

    get: mark/student/{studentId} all 3 spl marks of a student 
        add query param {splName} mark of a student for a specific spl

    post: mark/presentation-mark/ body {markId, presentationId, mark}
    put: mark/presentation-mark/ body{presentationMarkId, mark}
    get: mark/presentation-mark/ {query params: optional, splName, teacherId, presentationId}

    post: mark/continuous-mark/ body{markId, weekName, mark}
    put: mark/continuous-mark/ body{continuousMarkId, mark}

6. Presentation Event
    post: presentation/
    put: presentation/

7. Notice 
    post: notice/
    get: notice/ 
    get: notice/{noticeId}
    put: notice/ body{noticeId, notice}

8. Notification
    post: notification/
    get: notification/
    get: notification/{notificationId}
    put: notification/ body{notificationId, notification}
